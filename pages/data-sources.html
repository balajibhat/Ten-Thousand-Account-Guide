BI Dashboard

The centralized reporting dashboard for Ten Thousand performance data.

Lovable BI Dashboard Live
URL tt-bi-dashboard.lovable.app
Architecture Pulls directly from platform APIs (Google Ads, Meta, Snap). No ETL layer yet — data flows straight from the APIs to the dashboard. This means the dashboard reflects the most recent available data from each platform, subject to each platform's own reporting lag.
Core vs Blended pCPA The dashboard's primary reporting split is Core pCPA vs Blended pCPA. This split is driven entirely by the [CORE] / [EXP] naming convention in campaign and ad set names. The dashboard parses names to classify each campaign/ad set. See the Campaign Buckets page for full naming convention documentation.
Naming dependency If a campaign or ad set is misnamed (missing or incorrect [CORE]/[EXP] tag), it will be miscategorized in the dashboard. Naming accuracy is a reporting accuracy dependency. When graduating campaigns from [EXP] to [CORE], the dashboard automatically recalculates once the name is updated in the platform.
